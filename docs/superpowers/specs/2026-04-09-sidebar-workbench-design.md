# Sidebar Workbench Design

## 背景

当前 `Code Info` 侧边栏直接复用了 Dashboard 的 compact 版结构。虽然功能完整，但视觉和交互仍然像“缩小后的详情页”，不够适合作为个人开发时常驻的工作台。

这会带来几个问题：

- 侧边栏信息层级过深，第一眼看不到最重要的结论
- 大量面板式布局在窄宽度下显得拥挤
- 高价值动作没有形成稳定的操作区
- 侧边栏和详情页职责重叠，区分度不够

## 目标

把侧边栏重构成一个更适合个人开发者日常驻留的 `工作台`：

- 一眼知道当前范围和当前仓库状态
- 快速看到今天值得先看的内容
- 常用动作能在 1 次点击内完成
- 详情页继续承担完整分析和深度浏览

## 非目标

这一期不做：

- 详情页整体布局重构
- 数据模型大改
- 新增后端分析能力
- 把所有表格都搬进侧边栏

## 设计方向

侧边栏改成 3 段式结构：

### 1. 当前状态

顶部显示：

- 当前统计范围
- 当前 Git 仓库（多根工作区时）
- 最近更新时间
- 一条主结论文案

主结论文案优先从范围统计生成，例如：

- `今天改了 8 个文件，新增 2 个`
- `最近 7 天改了 21 个文件，TODO 3 个`

这里不再复用详情页的 topbar + chips 组合，而是用更紧凑的摘要卡。

### 2. 今日该看什么

中间区域只保留 3 组高价值内容：

- `变更概览`
- `待办热点`
- `快速入口`

其中：

- 变更概览展示少量关键数字和 3 条以内的重点文件
- 待办热点展示 3 条以内热点文件或待办位置
- 快速入口展示 `变更对比`、`详情分析`、`项目分析`、`切换 Git 仓库`

这一区域强调“下一步该点哪里”，而不是完整浏览。

### 3. 项目速读

底部展示项目级的少量长期指标：

- 主力语言
- 核心目录
- 项目文件数
- Git 趋势摘要

只保留摘要，不渲染大图表和大表格。

## 布局原则

- 侧边栏不再使用详情页的 `topbar + main panels` 视觉结构
- 所有模块都按窄列优化，避免横向拥挤
- 每一组内容都应该能在 280~360px 宽度下自然阅读
- 列表和指标卡要短，不堆大块表格

## 运行时策略

保留现有 `dashboard.js` 作为统一 runtime，但在 `presentation.compact === true` 时走独立的 sidebar workbench 渲染分支。

这样可以：

- 继续复用现有数据 payload
- 不影响详情页 runtime
- 把差异集中在 compact 分支中

## 实现边界

主要改动范围：

- `src/ui/sidebar.ts`
- `src/webview/templates.ts`
- `media/webview/dashboard.js`
- `media/webview/macos26.css`
- `src/test/extension.test.ts`

## 验收标准

- 侧边栏视觉结构明显区别于详情页
- 侧边栏默认不再出现详情页式的大段 panel 流布局
- 侧边栏能稳定展示：
  - 当前范围
  - 当前 Git 仓库
  - 关键变更摘要
  - 高价值快捷操作
- 详情页布局和交互保持不变
